Interpolating Functions and Expressions Within Strings

Problem

You want to create string using a mixture of literals and computed values. For example you want to display the current date is the string "Today's date is "

Solution

Using the quasistring library (from http://people.cs.uchicago.edu/~jacobm/qstr/) you can write code like: 

(require (lib "quasistring.ss" "quasistring"))
(require (lib "date.ss"))
(define today
  (let* ((seconds (current-seconds))
         (today (seconds->date seconds)))
    (date->string today)))
(qs "Today's date is $today")

You can also mix expressions and code with quasistrings: 

(require (lib "quasistring.ss" "quasistring"))
(require (lib "date.ss"))
(define today
  (let* ((seconds (current-seconds))
     (today (seconds->date seconds)))
    (date->string today)))
(qs "Today's date is $(let* ((seconds (current-seconds))
                             (today (seconds->date seconds)))
                        (date->string today)))")

Discussion

Although convenient, the quasistring notation is not considered Schemely by many. An alternative is to make use of Scheme's generic display procedure and variable argument list: 

(define (qs . args)
  (let ((port (open-output-string)))
    (for-each (lambda (arg) (display arg port)) args)
    (get-output-string port)))
(qs "Today's date is " (date->string (seconds->date (current-seconds))))
